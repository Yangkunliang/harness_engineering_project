# 编码规范 - 基于阿里巴巴Java开发手册（嵩山版）

> 本规范为项目编码的强制性标准，所有代码必须遵守

---

## 1. 命名规范

### 1.1 包名
- 全部小写，使用英文句点分隔
- 反向域名：`com.example.app.{module}`

### 1.2 类名
- PascalCase（大驼峰）
- 名词或名词短语
- 接口名以 `I` 开头（如 `IUserService`）
- 抽象类名以 `Abstract` 开头（如 `AbstractService`）

### 1.3 方法名
- camelCase（小驼峰）
- 动词或动词短语
- 查询用 `get/find/query`，删除用 `delete/remove`，更新用 `update`，保存用 `save/create`

### 1.4 变量名
- camelCase（小驼峰）
- 见名知意，禁止使用拼音缩写
- 布尔类型变量以 `is/has/can` 开头（如 `isEnabled`, `hasPermission`）

### 1.5 常量名
- UPPER_SNAKE_CASE（大写下划线）
- 静态常量必须定义在 `Constant` 类中

## 2. 格式规范

### 2.1 缩进
- 使用4个空格，禁止使用Tab

### 2.2 换行
- 一行代码不超过120字符
- 运算符前换行
- 方法参数超过3个时，每行一个参数

### 2.3 空格
- 二元运算符两侧必须有空格
- 方法名与括号之间无空格
- 逗号后必须有空格

### 2.4 大括号
- 左大括号不换行（K&R风格）
- 右大括号单独占一行

---

## 3. 编程规范

### 3.1 空值处理
- 禁止直接使用 `null`，使用 `Optional` 或空集合
- 所有入参必须进行非空校验
- 使用 `Objects.requireNonNull()` 进行参数校验

### 3.2 异常处理
- 禁止捕获 `Exception` 大类，必须捕获具体异常
- 禁止吞掉异常（空的catch块）
- 自定义异常必须继承 `RuntimeException`

### 3.3 集合操作
- 初始化集合时指定初始容量
- 使用增强for循环，禁止使用下标遍历
- 禁止在遍历过程中修改集合

### 3.4 日志规范
- 使用 `SLF4J` 日志框架
- 日志级别：DEBUG（调试）、INFO（业务流程）、WARN（警告）、ERROR（错误）
- 禁止在循环中打印日志

### 3.5 资源管理
- 使用 try-with-resources 自动关闭资源
- 禁止手动管理资源生命周期

## 4. 设计规范

### 4.1 分层架构
```
Controller → Service → Repository → Database
```
- Controller：处理HTTP请求，参数校验
- Service：业务逻辑处理
- Repository：数据访问

### 4.2 依赖注入
- 使用构造器注入，禁止字段注入
- 依赖必须声明为 `final`

### 4.3 DTO模式
- 请求参数使用 `RequestDTO`
- 响应数据使用 `ResponseDTO`
- 禁止直接返回实体类

### 4.4 错误码规范
| 错误码 | 含义 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权 |
| 403 | 禁止访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

## 5. 注释规范

### 5.1 类注释
- 必须包含类的功能说明
- 必须包含作者信息
- 必须包含创建日期

### 5.2 方法注释
- 必须包含功能说明
- 必须包含参数说明（`@param`）
- 必须包含返回值说明（`@return`）
- 必须包含异常说明（`@throws`）

### 5.3 代码注释
- 复杂逻辑必须有注释
- 注释必须与代码同步更新
- 禁止过时注释

---

## 6. 安全规范

### 6.1 SQL注入防护
- 使用预编译语句（PreparedStatement）
- 使用JPA/MyBatis的参数化查询
- 禁止拼接SQL字符串

### 6.2 XSS防护
- 对用户输入进行HTML转义
- 使用安全的模板引擎

### 6.3 敏感信息保护
- 禁止日志打印敏感信息
- 密码使用BCrypt加密存储
- 使用环境变量管理敏感配置
