# 业务模型

> 定义项目的核心业务实体和业务规则

---

## 一、核心业务实体

### 1. 用户（User）

**业务含义**：系统的注册用户

**属性**：
| 属性名 | 类型 | 含义 | 约束 |
|--------|------|------|------|
| id | Long | 用户唯一标识 | 主键 |
| username | String | 用户名 | 唯一，长度3-50 |
| password | String | 密码（加密后） | 非空 |
| email | String | 邮箱 | 唯一，格式校验 |
| phone | String | 手机号 | 可选 |
| nickname | String | 昵称 | 可选 |
| status | Integer | 状态 | 0=禁用，1=启用 |
| createdAt | LocalDateTime | 创建时间 | 非空 |
| updatedAt | LocalDateTime | 更新时间 | 非空 |

**业务规则**：
- 用户注册需验证邮箱唯一性
- 用户密码使用BCrypt加密存储
- 用户可通过邮箱或手机号登录

---

### 2. 商品（Product）

**业务含义**：商城中的商品

**属性**：
| 属性名 | 类型 | 含义 | 约束 |
|--------|------|------|------|
| id | Long | 商品唯一标识 | 主键 |
| name | String | 商品名称 | 非空，长度1-200 |
| description | String | 商品描述 | 可选 |
| price | BigDecimal | 价格 | 非空，≥0 |
| stock | Integer | 库存数量 | 非空，≥0 |
| categoryId | Long | 分类ID | 外键 |
| imageUrl | String | 图片URL | 可选 |
| status | Integer | 状态 | 0=下架，1=上架 |
| createdAt | LocalDateTime | 创建时间 | 非空 |
| updatedAt | LocalDateTime | 更新时间 | 非空 |

**业务规则**：
- 商品价格不能为负数
- 库存不足时无法下单
- 下架商品无法购买

---

### 3. 订单（Order）

**业务含义**：用户的购买订单

**属性**：
| 属性名 | 类型 | 含义 | 约束 |
|--------|------|------|------|
| id | Long | 订单唯一标识 | 主键 |
| orderNo | String | 订单编号 | 唯一，格式：YYYYMMDD+8位随机数 |
| userId | Long | 用户ID | 外键 |
| totalAmount | BigDecimal | 订单总额 | 非空，≥0 |
| status | Integer | 订单状态 | 见状态定义 |
| payStatus | Integer | 支付状态 | 0=未支付，1=已支付 |
| shippingAddress | String | 收货地址 | 非空 |
| createdAt | LocalDateTime | 创建时间 | 非空 |
| paidAt | LocalDateTime | 支付时间 | 可选 |
| shippedAt | LocalDateTime | 发货时间 | 可选 |
| completedAt | LocalDateTime | 完成时间 | 可选 |

**订单状态流转**：
```
待支付 → 待发货 → 待收货 → 已完成
     ↓
  已取消
```

**业务规则**：
- 订单创建后30分钟未支付自动取消
- 订单完成后可申请退款（7天内）
- 订单金额 = Σ(商品单价 × 数量)

---

### 4. 订单商品（OrderItem）

**业务含义**：订单中的商品明细

**属性**：
| 属性名 | 类型 | 含义 | 约束 |
|--------|------|------|------|
| id | Long | 明细唯一标识 | 主键 |
| orderId | Long | 订单ID | 外键 |
| productId | Long | 商品ID | 外键 |
| quantity | Integer | 购买数量 | 非空，≥1 |
| price | BigDecimal | 购买单价 | 非空，≥0 |

**业务规则**：
- 订单明细数量必须≥1
- 购买单价为下单时的商品价格

---

### 5. 商品分类（Category）

**业务含义**：商品分类

**属性**：
| 属性名 | 类型 | 含义 | 约束 |
|--------|------|------|------|
| id | Long | 分类唯一标识 | 主键 |
| name | String | 分类名称 | 非空，唯一 |
| parentId | Long | 父分类ID | 可选（顶级分类为null） |
| sortOrder | Integer | 排序顺序 | 默认0 |
| status | Integer | 状态 | 0=禁用，1=启用 |

**业务规则**：
- 分类支持三级树形结构
- 子分类继承父分类状态

---

## 二、核心业务流程

### 1. 购物流程

```
浏览商品 → 加入购物车 → 确认订单 → 支付 → 发货 → 收货 → 完成
```

### 2. 订单流程

```
创建订单 → 支付订单 → 商家发货 → 用户确认收货 → 订单完成
     ↓
  取消订单（未支付时）
```

### 3. 退款流程

```
申请退款 → 审核 → 退款成功
     ↓
  退款失败
```

## 三、业务规则汇总

| 规则编号 | 规则描述 | 关联实体 |
|----------|----------|----------|
| BR-001 | 用户密码必须加密存储 | User |
| BR-002 | 订单创建后30分钟未支付自动取消 | Order |
| BR-003 | 库存不足时无法下单 | Product, Order |
| BR-004 | 下架商品无法购买 | Product |
| BR-005 | 订单完成后7天内可申请退款 | Order |
| BR-006 | 分类支持三级树形结构 | Category |
