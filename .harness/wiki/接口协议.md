# 接口协议

> 定义REST API接口规范和协议

---

## 一、基础协议

### 1. 协议类型
- **协议**: HTTPS
- **字符编码**: UTF-8
- **数据格式**: JSON

### 2. 通用请求头
| Header | 值 | 说明 |
|--------|-----|------|
| Content-Type | application/json | 请求体格式 |
| Accept | application/json | 响应格式 |
| Authorization | Bearer {token} | 认证令牌（登录后） |

### 3. 通用响应格式

**成功响应**：
```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1704067200000
}
```

**失败响应**：
```json
{
  "code": 400,
  "message": "请求参数错误",
  "data": null,
  "timestamp": 1704067200000
}
```

**分页响应**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "content": [],
    "totalElements": 100,
    "totalPages": 10,
    "currentPage": 1,
    "pageSize": 10
  },
  "timestamp": 1704067200000
}
```

### 4. 错误码定义

| 错误码 | 含义 | HTTP状态码 |
|--------|------|------------|
| 200 | 成功 | 200 |
| 400 | 请求参数错误 | 400 |
| 401 | 未授权（未登录） | 401 |
| 403 | 禁止访问（无权限） | 403 |
| 404 | 资源不存在 | 404 |
| 500 | 服务器内部错误 | 500 |
| 1001 | 用户不存在 | 404 |
| 1002 | 密码错误 | 400 |
| 1003 | 用户已存在 | 400 |
| 2001 | 商品不存在 | 404 |
| 2002 | 商品已下架 | 400 |
| 2003 | 库存不足 | 400 |
| 3001 | 订单不存在 | 404 |
| 3002 | 订单状态不允许此操作 | 400 |

## 二、用户接口

### 1. 用户注册

**路径**: `POST /api/users/register`

**请求体**:
```json
{
  "username": "string (必填，3-50字符)",
  "password": "string (必填，6-50字符)",
  "email": "string (必填，邮箱格式)",
  "phone": "string (可选，手机号格式)",
  "nickname": "string (可选)"
}
```

**成功响应**:
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "id": 1,
    "username": "user001",
    "email": "user@example.com",
    "nickname": "用户昵称",
    "status": 1,
    "createdAt": "2024-01-01T12:00:00"
  }
}
```

---

### 2. 用户登录

**路径**: `POST /api/users/login`

**请求体**:
```json
{
  "username": "string (必填，用户名或邮箱)",
  "password": "string (必填)"
}
```

**成功响应**:
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 1,
      "username": "user001",
      "email": "user@example.com"
    }
  }
}
```

---

### 3. 获取用户信息

**路径**: `GET /api/users/me`

**认证**: 需要Authorization头

**成功响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "username": "user001",
    "email": "user@example.com",
    "phone": "13800138000",
    "nickname": "用户昵称",
    "status": 1,
    "createdAt": "2024-01-01T12:00:00"
  }
}
```

## 三、商品接口

### 1. 获取商品列表

**路径**: `GET /api/products`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| page | Integer | 否 | 页码，默认1 |
| size | Integer | 否 | 每页数量，默认10 |
| categoryId | Long | 否 | 分类ID筛选 |
| keyword | String | 否 | 关键词搜索 |

**成功响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "content": [
      {
        "id": 1,
        "name": "商品名称",
        "description": "商品描述",
        "price": 99.99,
        "stock": 100,
        "categoryId": 1,
        "imageUrl": "https://example.com/image.jpg",
        "status": 1
      }
    ],
    "totalElements": 100,
    "totalPages": 10,
    "currentPage": 1,
    "pageSize": 10
  }
}
```

---

### 2. 获取商品详情

**路径**: `GET /api/products/{id}`

**成功响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "name": "商品名称",
    "description": "商品描述",
    "price": 99.99,
    "stock": 100,
    "categoryId": 1,
    "categoryName": "分类名称",
    "imageUrl": "https://example.com/image.jpg",
    "status": 1,
    "createdAt": "2024-01-01T12:00:00"
  }
}
```

---

### 3. 创建商品（管理员）

**路径**: `POST /api/products`

**认证**: 需要管理员权限

**请求体**:
```json
{
  "name": "string (必填，1-200字符)",
  "description": "string (可选)",
  "price": "number (必填，≥0)",
  "stock": "integer (必填，≥0)",
  "categoryId": "long (必填)",
  "imageUrl": "string (可选)"
}
```

## 四、订单接口

### 1. 创建订单

**路径**: `POST /api/orders`

**认证**: 需要登录

**请求体**:
```json
{
  "items": [
    {
      "productId": "long (必填)",
      "quantity": "integer (必填，≥1)"
    }
  ],
  "shippingAddress": "string (必填)"
}
```

**成功响应**:
```json
{
  "code": 200,
  "message": "订单创建成功",
  "data": {
    "id": 1,
    "orderNo": "2024010112345678",
    "totalAmount": 199.98,
    "status": 0,
    "payStatus": 0,
    "shippingAddress": "收货地址",
    "createdAt": "2024-01-01T12:00:00"
  }
}
```

---

### 2. 获取订单列表

**路径**: `GET /api/orders`

**认证**: 需要登录

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| page | Integer | 否 | 页码，默认1 |
| size | Integer | 否 | 每页数量，默认10 |
| status | Integer | 否 | 订单状态筛选 |

---

### 3. 获取订单详情

**路径**: `GET /api/orders/{id}`

**认证**: 需要登录，只能查看自己的订单

**成功响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "orderNo": "2024010112345678",
    "totalAmount": 199.98,
    "status": 1,
    "payStatus": 1,
    "shippingAddress": "收货地址",
    "items": [
      {
        "productId": 1,
        "productName": "商品名称",
        "quantity": 2,
        "price": 99.99
      }
    ],
    "createdAt": "2024-01-01T12:00:00",
    "paidAt": "2024-01-01T12:05:00"
  }
}
```

## 六、接口权限矩阵

| 接口路径 | 权限要求 |
|----------|----------|
| POST /api/users/register | 无 |
| POST /api/users/login | 无 |
| GET /api/users/me | 已登录 |
| GET /api/products | 无 |
| GET /api/products/{id} | 无 |
| POST /api/products | 管理员 |
| PUT /api/products/{id} | 管理员 |
| DELETE /api/products/{id} | 管理员 |
| POST /api/orders | 已登录 |
| GET /api/orders | 已登录 |
| GET /api/orders/{id} | 已登录（自己的订单） |
| PUT /api/orders/{id}/pay | 已登录 |
| PUT /api/orders/{id}/cancel | 已登录 |
| GET /api/categories | 无 |
| POST /api/categories | 管理员 |
