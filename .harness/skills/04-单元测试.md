# 单元测试 Skill

> 覆盖率≥80%，核心100%

---

## 输入

| 输入项 | 类型 | 说明 |
|--------|------|------|
| 源代码 | 源代码 | 需要测试的代码 |
| 测试框架 | 工具 | JUnit 5 + Mockito |

## 步骤

### 步骤1：确定测试范围

1. 核心业务逻辑必须100%覆盖
2. 辅助功能至少80%覆盖
3. 边界条件必须测试

### 步骤2：编写测试用例

| 测试类型 | 说明 | 示例 |
|----------|------|------|
| 正常流程 | 测试正常业务流程 | 用户注册成功 |
| 异常流程 | 测试异常情况 | 参数校验失败 |
| 边界条件 | 测试边界值 | 空输入、最大长度 |
| 并发测试 | 测试并发场景 | 库存扣减 |

### 步骤3：执行测试

1. 运行单元测试
2. 生成覆盖率报告
3. 分析未覆盖代码

### 步骤4：修复缺陷

1. 根据测试结果修复bug
2. 补充缺失的测试用例
3. 重新执行测试

## 覆盖率要求

| 指标 | 要求 |
|------|------|
| 整体覆盖率 | ≥80% |
| 核心模块 | 100% |
| 分支覆盖率 | ≥70% |

## 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 测试报告 | 文档 | 测试执行结果 |
| 覆盖率报告 | 文档 | 代码覆盖情况 |

## 检查点

- [ ] 单元测试全部通过
- [ ] 整体覆盖率≥80%
- [ ] 核心模块覆盖率100%
