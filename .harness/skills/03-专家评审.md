# 专家评审 Skill

> 6维评审、阻断机制

---

## 输入

| 输入项 | 类型 | 说明 |
|--------|------|------|
| 代码变更 | 源代码 | 需要评审的代码 |
| 技术方案 | 文档 | 实现方案说明 |
| 测试报告 | 文档 | 单元测试结果 |

## 6维评审

### 1. 架构维度

| 检查项 | 说明 | 通过/阻断 |
|--------|------|-----------|
| 分层原则 | 是否符合 Controller → Service → Repository | |
| 依赖关系 | 是否存在循环依赖 | |
| 设计模式 | 是否合理使用设计模式 | |

### 2. 安全维度

| 检查项 | 说明 | 通过/阻断 |
|--------|------|-----------|
| 认证授权 | 是否有正确的权限控制 | |
| SQL注入 | 是否使用参数化查询 | |
| 敏感信息 | 是否有敏感信息泄露风险 | |

### 3. 性能维度

| 检查项 | 说明 | 通过/阻断 |
|--------|------|-----------|
| 查询优化 | 是否有N+1查询问题 | |
| 缓存策略 | 是否合理使用缓存 | |
| 并发处理 | 是否考虑并发安全 | |

### 4. 可维护维度

| 检查项 | 说明 | 通过/阻断 |
|--------|------|-----------|
| 代码可读性 | 命名是否清晰、注释是否充分 | |
| 模块化 | 是否符合单一职责原则 | |
| 复杂度 | 方法复杂度是否过高 | |

### 5. 规范维度

| 检查项 | 说明 | 通过/阻断 |
|--------|------|-----------|
| 编码规范 | 是否符合阿里巴巴Java开发手册 | |
| 命名规范 | 是否符合项目命名约定 | |
| 格式规范 | 是否符合代码格式要求 | |

### 6. 测试维度

| 检查项 | 说明 | 通过/阻断 |
|--------|------|-----------|
| 测试覆盖 | 单元测试覆盖率是否达标 | |
| 测试质量 | 测试用例是否有效 | |
| 集成测试 | 是否有必要的集成测试 | |

## 阻断机制

- **任意维度不通过** → 阻断合并
- **建议改进项** → 记录并跟踪

## 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 评审报告 | 文档 | 评审结果汇总 |
| 改进建议 | 文档 | 需要修复的问题 |

## 检查点

- [ ] 所有维度评审通过
- [ ] 改进建议已记录
