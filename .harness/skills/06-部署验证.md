## 输入

| 输入项 | 类型 | 说明 |
|--------|------|------|
| CI通过的代码 | 部署包 | 经过CI验证的代码 |
| 部署环境 | 服务器 | 测试/生产环境 |

## 验证步骤

### 步骤1：部署到测试环境

**命令**：
```bash
# 使用Docker Compose一键部署
docker-compose up --build -d

# 等待服务启动
sleep 30
```

### 步骤2：冒烟测试

| 检查项 | 说明 | 命令 | 通过/失败 |
|--------|------|------|-----------|
| 服务启动 | 应用是否正常启动 | `docker-compose ps` | |
| 健康检查 | `/actuator/health` 是否返回UP | `curl -s http://localhost:8080/actuator/health | grep UP` | |
| 前端访问 | 首页是否可访问 | `curl -s http://localhost:5173 | head -1` | |
| 核心API | 用户注册接口是否可用 | `curl -s -X POST http://localhost:8080/api/auth/register -H "Content-Type: application/json" -d '{"username":"test","password":"123456","email":"test@example.com"}' | grep success` | |

**自动化脚本**：
```bash
#!/bin/bash
echo "=== 冒烟测试 ==="

# 检查后端服务
echo ">>> 检查后端服务"
curl -s http://localhost:8080/actuator/health | grep -q "UP" || exit 1
echo "✓ 后端健康检查通过"

# 检查前端服务
echo ">>> 检查前端服务"
curl -s http://localhost:5173 | grep -q "<title>" || exit 1
echo "✓ 前端服务正常"

echo "=== 冒烟测试通过 ==="
```

| 消息消费 | 是否能正常消费消息 | |

### 步骤4：Redis验证

| 检查项 | 说明 | 通过/失败 |
|--------|------|-----------|
| 连接测试 | Redis连接是否正常 | |

### 步骤5：性能测试

**命令**（使用wrk）：
```bash
# 安装wrk
brew install wrk

# 性能测试
wrk -t4 -c100 -d30s http://localhost:8080/api/products

# 结果分析
# 要求：Latency < 200ms, Requests/sec > 1000
```

| 指标 | 要求 | 实际值 |
|------|------|--------|
| 响应时间 | < 200ms | |
| QPS | > 1000 | |
| 错误率 | < 0.1% | |
| CPU使用率 | < 70% | |
| 内存使用率 | < 80% | |

## 一键部署验证脚本

```bash
#!/bin/bash
echo "=== 部署验证自动化 ==="

# 步骤1：部署服务
echo ">>> 步骤1：部署服务"
docker-compose up --build -d
sleep 30

# 步骤2：冒烟测试
echo ">>> 步骤2：冒烟测试"
curl -s http://localhost:8080/actuator/health | grep -q "UP" || { echo "✗ 后端健康检查失败"; exit 1; }
echo "✓ 后端健康检查通过"

curl -s http://localhost:5173 | grep -q "<title>" || { echo "✗ 前端服务异常"; exit 1; }
echo "✓ 前端服务正常"

# 步骤3：API测试
echo ">>> 步骤3：API测试"
curl -s -X GET http://localhost:8080/api/products | grep -q "content" || { echo "✗ 商品API异常"; exit 1; }
echo "✓ 商品API正常"

echo "=== 部署验证通过！==="
```
