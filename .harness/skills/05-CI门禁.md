# CI门禁 Skill

> 6道关卡，全过才放行

---

## 输入

| 输入项 | 类型 | 说明 |
|--------|------|------|
| 代码变更 | 源代码 | 需要验证的代码 |
| CI配置 | 配置 | Jenkins/GitLab CI配置 |

## 6道关卡

### 关卡1：编译检查

- **检查内容**：Maven编译是否成功
- **工具**：Maven
- **命令**：
  ```bash
  # 后端编译
  cd backend && mvn clean compile -q
  
  # 前端构建
  cd frontend && npm run build
  ```
- **失败条件**：编译错误

### 关卡2：规范检查

- **检查内容**：代码是否符合编码规范
- **工具**：Checkstyle + ESLint
- **命令**：
  ```bash
  # 后端规范检查
  cd backend && mvn checkstyle:check
  
  # 前端规范检查
  cd frontend && npx eslint . --quiet
  ```
- **失败条件**：规范违规

### 关卡3：测试检查

- **检查内容**：单元测试和E2E测试是否通过
- **工具**：JUnit 5 + Playwright
- **命令**：
  ```bash
  # 后端单元测试
  cd backend && mvn test -q
  
  # 前端E2E测试（需启动开发服务器）
  cd frontend && npm run test:e2e
  ```
- **失败条件**：任意测试失败

### 关卡4：覆盖率检查

- **检查内容**：测试覆盖率是否达标
- **工具**：JaCoCo
- **命令**：
  ```bash
  cd backend && mvn jacoco:report
  ```
- **失败条件**：覆盖率低于80%

### 关卡5：依赖检查

- **检查内容**：是否有安全漏洞依赖
- **工具**：OWASP Dependency Check
- **命令**：
  ```bash
  cd backend && mvn org.owasp:dependency-check-maven:check
  ```
- **失败条件**：存在高危漏洞

### 关卡6：安全检查

- **检查内容**：代码安全问题扫描
- **工具**：SonarQube
- **命令**：
  ```bash
  cd backend && mvn sonar:sonar \
    -Dsonar.projectKey=harness-ecommerce \
    -Dsonar.host.url=http://localhost:9000 \
    -Dsonar.login=admin \
    -Dsonar.password=admin
  ```
- **失败条件**：存在严重安全问题

## 一键自动化脚本

```bash
#!/bin/bash
echo "=== CI门禁自动化检查 ==="

# 关卡1：编译检查
echo ">>> 关卡1：编译检查"
cd backend && mvn clean compile -q || exit 1
cd ../frontend && npm run build || exit 1

# 关卡2：规范检查
echo ">>> 关卡2：规范检查"
cd ../backend && mvn checkstyle:check || exit 1
cd ../frontend && npx eslint . --quiet || exit 1

# 关卡3：测试检查
echo ">>> 关卡3：测试检查"
cd ../backend && mvn test -q || exit 1
cd ../frontend && npm run test:e2e || exit 1

# 关卡4：覆盖率检查
echo ">>> 关卡4：覆盖率检查"
cd ../backend && mvn jacoco:report || exit 1

echo "=== 所有关卡通过！==="
```

## 门禁规则

- **全过才放行**：任意关卡失败则CI失败
- **自动回滚**：失败后自动取消部署
- **通知机制**：失败后通知相关人员
