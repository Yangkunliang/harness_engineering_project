# 需求分析 Skill

> 识别影响范围、冲突检测

---

## 输入

| 输入项 | 类型 | 说明 |
|--------|------|------|
| 需求文档 | 文档 | 产品需求说明 |
| 产品原型 | 设计稿 | UI/UX设计 |
| 当前系统状态 | 数据 | 现有系统信息 |

## 步骤

### 步骤1：需求拆解

1. 通读需求文档，理解业务目标
2. 识别核心功能点
3. 确定非功能需求（性能、安全、可用性）

### 步骤2：影响范围分析

| 维度 | 分析内容 |
|------|----------|
| 数据层 | 涉及哪些表、字段变更 |
| 业务层 | 影响哪些业务流程 |
| 接口层 | 需要新增/修改哪些API |
| 权限层 | 是否涉及权限变更 |

### 步骤3：冲突检测

- [ ] 是否与现有需求冲突
- [ ] 是否与技术架构冲突
- [ ] 是否有资源冲突（人力、时间）

### 步骤4：技术方案设计

1. 确定技术实现方案
2. 评估可行性和复杂度
3. 制定实施计划

## 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 需求分析报告 | 文档 | 需求拆解结果 |
| 影响范围报告 | 文档 | 各层影响分析 |
| 技术方案文档 | 文档 | 实现方案 |

## 检查点

- [ ] 需求已清晰理解
- [ ] 影响范围已明确
- [ ] 无需求冲突
- [ ] 技术方案可行
